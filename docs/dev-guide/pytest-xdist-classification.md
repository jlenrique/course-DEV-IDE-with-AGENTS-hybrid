# Pytest Xdist Classification

## Background

Story 7c.0c implements AMEND-V1 and AMEND-V2 from
`_bmad-output/planning-artifacts/slab-7c-velocity-amendments-2026-05-04.md`.
The goal is to reduce the default broad-regression wall-clock while preserving
the NFR-7c-R2 deterministic baseline. The T1 serial baseline on this workspace
was `39 failed, 4041 passed, 27 skipped, 46 deselected, 2 xfailed` in 419.34s.
The first xdist diagnostic was faster, but produced two xdist-only failures.

## Classification Methodology

Run the serial baseline first, then run:

```bash
.venv/Scripts/python.exe -m pytest -n auto --dist loadfile -p no:randomly -q --tb=line
```

Only tests that pass serial and fail under xdist receive `@pytest.mark.serial`.
Do not mark a test because it looks stateful. Classify each empirical offender
into one bucket: DB-shared-state, Filesystem collisions, Port-binding,
LangSmith trace coupling, Random-seed coupling, or Other.

## Project-Default Invocation

The canonical pytest config is `pyproject.toml::[tool.pytest.ini_options]`.
The default `addopts` is:

```toml
addopts = "-p no:randomly -n auto --dist loadfile -m 'not live and not serial'"
```

Use these commands:

```bash
# Parallel default pass.
.venv/Scripts/python.exe -m pytest -p no:randomly -q --tb=line

# Serial-marked subset.
.venv/Scripts/python.exe -m pytest -n0 -p no:randomly -m "serial" -q --tb=line

# R2 smoke pass from the JSON manifest.
.venv/Scripts/python.exe -m pytest --smoke -p no:randomly -q --tb=short

# Full serial diagnostic if xdist behavior needs re-baselining.
.venv/Scripts/python.exe -m pytest -n0 -p no:randomly -q --tb=line
```

Pytest's native `@file` parser treats each line as one CLI argument and cannot
parse a JSON array. The repository therefore keeps the authoritative manifest
as JSON and exposes the zero-dependency `--smoke` selector in `tests/conftest.py`.

## Smoke-Suite Manifest

`tests/_smoke_suite_manifest.json` is the AMEND-V2 R2 manifest. It contains 200
pytest node IDs and is regenerated by:

```bash
.venv/Scripts/python.exe scripts/utilities/curate_smoke_suite.py
```

The initial manifest covers TripwireLedgerEntry and audit-chain shape,
retrieval intent and Texas handoff contracts, gate shim behavior, composition
precedence, schema validators, and activation-contract smoke tests. Re-curate at
each Slab 7c wave close so new load-bearing tests enter the R2 tier.

## Maintenance

New tests should default to parallel-safe. Add `@pytest.mark.serial` only after a
serial-vs-xdist diagnostic proves the test passes serial and fails under xdist.
Update this registry in the same change as the marker. At wave close, regenerate
the smoke manifest, run `pytest --smoke`, and update the `.meta` provenance file.
