## Scout ground truth (confirmed in code)

- Studio completeness requires `studio_template.gamma_id` but does NOT forbid a stray `theme.id` on a studio record → **from-template⊕theme_id is a real gap.**

## The build (ratified rules — Gary's Gamma-truth binding)
1. **Rule 2 — `image_model`⇒`aiGenerated` (ERROR/hard-block).** On the resolved surface: if `image_model` is present AND `image_source != "aiGenerated"`, the model string is a silent no-op → error `gamma.dep.image-model-source`. (Gary: accurate — model ignored under unsplash/webAllImages/giphy/pictographic.)
2. **Rule 3 — named `stylePreset` + conflicting `image_style` (WARN, raw-record, LOUD).** On the RAW record (resolution drops it): if `image_style_preset` is a **named preset** (not `custom`, not empty) AND `image_style` is also set, emit a **WARNING** `gamma.dep.preset-style-subordinated` — the preset dominates the card theme and the top-level `image_style` may be silently subordinated. **WARN not ERROR** (Gary: it's a merge, not invalid; a valid styleguide must not be blocked), but **surfaced loudly, never silently dropped** (Dan). Must stay consistent with the `gamma-keywords-to-imageoptions-style-channel` route (that channel survives).
3. **Rule 6 — `dimensions` Classic-only (DEDUP, membership assertion).** Do NOT add a second forbidden-set. Add a TEST that the existing discriminated-union/studio-surface check already rejects `dimensions` on a studio record (union-membership assertion). Only add a new check if a gap is found; if so, ONE message (no double-fire).
4. **NEW Rule — `from-template ⊕ theme_id` (ERROR/hard-block).** A studio (`production_mode=studio`) record must NOT also carry a non-empty `theme.id` — the template is the theme authority; both is a conflict Gamma resolves unpredictably → error `gamma.dep.studio-theme-conflict`. (Gary — high-value static pair; fills the union gap.)
5. **Narrow non-contradiction (WARN, if tractable) — fold from the filed follow-on.** ONLY the deterministic form: the styleguide's prose (`presentation.narrative.*` / `additional_instructions`) contains a literal enum token that contradicts a structured param (e.g. prose says "16:9 widescreen" while `dimensions` is `fluid`/absent; prose names a theme differing from `theme.id`). Pure string/enum, closed set, no LLM → WARN `gamma.dep.prose-param-contradiction`. If a given pair can't be made deterministic with a RED+good fixture, it stays out (graduates to the general follow-on). The general semantic validator stays filed (`gamma-prose-vs-param-noncontradiction-validator`).

## Acceptance criteria (RED-first; dual-gate)
- **AC#1 [warnings channel]** the validator gains a **warnings** list distinct from errors (WARN rules surface loudly but do not fail the exit code unless `--strict`); errors still fail exit non-zero. Pin the output shape.
- **AC#2 [Rule 2, ERROR]** a styleguide with `image_model` set + `image_source != aiGenerated` → error with tag/id `gamma.dep.image-model-source`; a good styleguide (aiGenerated + model) PASSES.
- **AC#3 [Rule 3, WARN raw-record]** a styleguide with a named `image_style_preset` + a set `image_style` → **WARN** (not error), diagnostic surfaced; a good styleguide (preset alone, OR custom+style) emits NO warning.
- **AC#4 [Rule 6, dedup]** a test asserts a studio record carrying `dimensions` is already rejected by the union/surface check (no new duplicate error); if a gap exists, one new error, no double-fire.
- **AC#5 [from-template⊕theme_id, ERROR]** a studio record with a non-empty `theme.id` → error `gamma.dep.studio-theme-conflict`; a good studio record (studio_template, no theme) PASSES; a good api record (theme, no studio_template) PASSES.
- **AC#6 [narrow non-contradiction WARN]** a styleguide whose prose literally contradicts a structured enum (deterministic pair) → WARN; a coherent styleguide emits none. (Drop the AC if no pair is deterministically tractable — record the decision.)
- **AC#7 [🔴 anti-false-block — Gary, load-bearing]** for EVERY rule above, a committed **known-GOOD styleguide fixture that PASSES** (no false ERROR, no spurious WARN). The 3 shipped seeds must remain validator-clean (0 errors) after all rules land.
- **AC#8 [hermetic no-network guard]** a test asserts the default `validate_style_guides(...)` invocation makes **no outbound network call** (the `--check-existence` live path stays flag-gated/off). Guards the boundary from silent regression.
- **AC#9 [teeth per rule, RED-first]** each ERROR/WARN rule has a committed **deliberately-misconfigured** fixture proven RED before the check exists; assert the **diagnostic id/string**, not just exit code.
