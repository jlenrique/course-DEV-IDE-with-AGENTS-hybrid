- `_enforce_verb_payload_consistency()` `model_validator(mode="after")` correctly enforces required-iff + forbids semantics for ALL THREE verbs:

  - `verb="edit"` → requires `edit_payload: PerSlideVariantEditPayload`; forbids `select_payload` and `reject_reason`.
  - `verb="reject"` → requires `reject_reason`; forbids `select_payload` and `edit_payload`.
- This is structurally **richer than §02A canonical** (2-way) and **isomorphic to 7c.14 §11** (3-way) — the closest-sibling pattern-replication target. The `select` verb carries operator-chosen `selected_variant: Literal["A","B"]` as REQUIRED state (not null-payload), so the positive `select_payload` constraint is correct.

Pattern-parity verified: NOT mistakenly authored as `["approve","edit","reject"]` (the §02A pattern). Correct divergence from §02A canonical, correct convergence with §11 (7c.14) sibling.
