## Why (product value / guardrail)
Give a styleguide a way to declare a **style-identity** property that is genuinely NOT expressible declaratively: "this style reads as a multi-beat narrative walk; below N beats it collapses to a stub deck." That intent is honored by feeding Irene's Pass-1 clustering a **minimum cluster floor**, improving VO **follow-along** on multi-beat styles (dense source chunked so narration can track it — [[project_chunking_via_clustering_followalong]]). PRODUCT-VALID: the floor must earn its place in follow-along/perception terms, **never** to make a proofing run produce a particular deck ([[feedback_spoc_is_goal_not_concierge_proofing_runs]]). **The floor proposes; the non-waivable 07G gate disposes.**
