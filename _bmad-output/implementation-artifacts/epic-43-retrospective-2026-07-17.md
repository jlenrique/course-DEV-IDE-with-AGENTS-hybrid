## Lessons

1. **The RED-first coverage ratchet is the durable fix, not the renderers.** A prose AC ("make it tabular") let 42-1 close on 1-of-15 surfaces. A test that enumerates every content type and fails until each is registered-or-explicitly-waived (with a shrink-only allowlist that 43-12 asserts empty) makes "closed on a subset" *mechanically impossible*. Every future gate must register a renderer or fail CI. This is the pattern to reuse for any "apply X to all N surfaces" requirement.
2. **The gate→content_type bridge (43-3) was the load-bearing seam.** 43-2 keyed by gate_id; the canonical keys are semantic. Establishing `GATE_TO_CONTENT_TYPE` once (keyed on the paused_gate string, with shared/folded gates deliberately unmapped + pinned) made 43-4…43-9 mechanical.
3. **Synthesize fixtures from the REAL model, not hand-authored JSON.** Every renderer's fixture was generated by invoking the real Pydantic card model / `display_*` function — shape fidelity by construction, and it surfaced real mismatches (see below).
4. **Honest de-scope beats phantom coverage.** research_packet/workbook aren't operator-reviewed gates; 43-9 removed them from the canonical set with rationale rather than building renderers nothing dispatches.
5. **Consumer-wide baseline-diff catches contract-wide escapes.** 43-1's change to `_confirm_or_edit_directive` tripped the TW-7c-4 scope-creep audit (new test files) and had to register them in `PERMITTED_PYTHON_DIFFS` — caught only because the orchestrator ran the *full* consumer set, not the dev agent's subset (the 42-3 lesson, re-confirmed).
6. **Push hygiene:** stacking concurrent background `git push` processes deadlocked the remote ref (15 zombie processes). One push at a time, wait for it. Reads stayed fast throughout — the tell.
