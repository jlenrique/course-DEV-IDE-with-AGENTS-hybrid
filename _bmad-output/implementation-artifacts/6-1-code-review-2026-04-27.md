**Not clean. Do not flip Slab 6.1 to `done`.**

## Triaged Findings

### Decision Needed

#### DN-1 - Probe-based runner vs actual production graph execution

**Sources:** Blind Hunter + Edge Case Hunter + Acceptance Auditor  
**Classification:** `decision_needed`  
**Affected:** `app/marcus/orchestrator/production_runner.py`, `app/manifest/compiler.py`  
**Evidence:** `run_production_trial()` calls `compile_run_graph(manifest)`, then iterates `manifest.nodes` and calls `_invoke_specialist_probe(...)` directly instead of invoking the compiled LangGraph or the registry-backed handler.  
**Why this needs operator decision:** A correct patch depends on the accepted definition of Slab 6.1:

1. **Strict AC interpretation:** production evidence requires executing the compiled LangGraph with registry-backed specialist handlers and real checkpointing. This likely needs a follow-on implementation slice, because several specialist graphs expect richer per-specialist state/tool inputs than the current probe supplies.
2. **Amended bounded interpretation:** Slab 6.1 is a production-launch probe layer that verifies manifest resolution plus one live OpenAI call, while full graph-stepping/checkpoint execution becomes Slab 6.2 or 6.x. This requires amending AC-A/B/G/H and keeping FR52 PARTIAL until the real graph execution path lands.

**Recommendation:** choose option 1. The A16 prevention purpose of this story is specifically composition exercise; accepting the probe path would preserve the same defect class.

#### DN-2 - HIL resume semantics are administrative, not graph-resume semantics

**Sources:** Blind Hunter + Edge Case Hunter + Acceptance Auditor  
**Classification:** `decision_needed`  
**Affected:** `app/marcus/orchestrator/production_runner.py`  
**Evidence:** Pause writes `checkpoint.json` with `next_node_index`; resume calls `resume_from_verdict()` and then marks the envelope `completed`. The returned LangGraph `Command` is written to `resume-command.json` but never passed back into a graph execution.  
**Why this needs operator decision:** If DN-1 is fixed with real LangGraph execution, this must become a LangGraph checkpoint/thread resume implementation. If DN-1 is amended to probe-only, resume-to-completion cannot honestly be claimed and AC-D must be amended.

**Recommendation:** couple this to DN-1 option 1 and require a downstream-node assertion after resume.

#### DN-3 - Evidence and LangSmith acceptance boundary

**Sources:** Blind Hunter + Edge Case Hunter + Acceptance Auditor  
**Classification:** `decision_needed`  
**Affected:** `app/marcus/orchestrator/production_runner.py`, `tests/integration/marcus/test_production_clone_launch_evidence_discipline.py`, `tests/live/test_production_trial_smoke.py`  
**Evidence:** Evidence becomes true when `OPENAI_API_KEY` exists, a probe call count is positive, and offline mode is false. Tests monkeypatch the chat model and still assert evidence true. Trace/cost measurement uses synthetic local trace objects.  
**Options:**

1. Require real graph execution plus real LangSmith trace/cost evidence before `production_clone_launch_evidence=True`.
2. Split the field into a weaker "live_probe_evidence" and reserve `production_clone_launch_evidence` for the full graph path.
3. Amend AC-F to accept a direct-probe definition, with explicit documentation that it does not close FR52.

**Recommendation:** option 1 or 2. Do not keep the current mocked-probe test as evidence=true.

#### DN-4 - M3 harness delta conflicts with the no-change constraint

**Sources:** Acceptance Auditor + local verification  
**Classification:** `decision_needed`  
**Affected:** `marcus/orchestrator/m3_trial.py`, `tests/fixtures/marcus/baseline_envelope/2026-04-26/envelope.json`  
**Evidence:** The diff changes `run_local_m3_trial()` to accept `trial_id` and `operator_id`, and changes the override model to `gpt-5-mini`. Reverting this file to `HEAD` breaks the current replay projection because the baseline fixture expects `gpt-5-mini`.  
**Options:**

1. Treat the M3 changes as pre-Slab-6.1 baseline drift and exclude them from Slab 6.1 review scope, documenting why they are not part of this story.
2. Revert M3 and regenerate/repair the replay baseline in a separate, explicit replay-baseline patch.
3. Keep M3 modified and amend the Slab 6.1 no-change constraint, which is not recommended because it weakens the frozen-harness rule.

**Recommendation:** option 1 only if operator confirms these changes predate Slab 6.1; otherwise option 2.
