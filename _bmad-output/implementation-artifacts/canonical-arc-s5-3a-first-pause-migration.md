## The migration principle (party record M1 — binding; refuse "both-worlds-green" coverage vanity)
Post-3b the **unset-env walk is canonical** and pauses at G0E→G0R→G1. 3a makes the 8 suites **robust to the flip WITHOUT depending on it** (3a must be green NOW, at default-OFF, and stay green after 3b). Per-test disposition (recorded in a **test-disposition ledger** in Completion Notes — `test → disposition → replacement witness`), one of:
- **(D-migrate-canonical)** the test's REAL subject is downstream of G0E (e.g. G1 card emission, resume/recover, gary parity): make it **env-explicit** — set `MARCUS_G0_ENRICHMENT_ACTIVE` to the value that yields the walk it means to test, and where it should exercise the new canonical path, cross G0E/G0R first (or set the kill-switch to reach G1 directly if G0-enrichment is irrelevant to that test's subject). The assertion becomes correct under BOTH env states by being explicit about which it drives.
- **(D-kill-switch-escape-hatch)** a test whose subject IS the G0-dormant path keeps `MARCUS_G0_ENRICHMENT_ACTIVE` explicitly unset/OFF and is RE-LABELED as escape-hatch/kill-switch coverage (comment says so) — NOT deleted, NOT "both worlds green" hedging.
- **(D-delete)** a test rendered genuinely redundant by the canonical G0E path — delete with a one-line ledger rationale + name its replacement witness (which lands in 3b).
**No test may silently rely on the default env value.** Every migrated walk sets the env it means to drive (explicit-OFF for kill-switch subjects; explicit-ON or unset-post-3b for canonical subjects). 3a introduces the explicitness; 3b flips the default and adds the canonical unset-env witness.
