**Date:** 2026-04-27

## Layer Findings

### Blind Hunter

- Adapter ignored dispatch-registry-backed builder refs and resolved specialists by naming convention.
- Adapter bypassed LangGraph execution by manually calling only `plan` and `act`.
- Missing-output detection could accept the preloaded input payload as specialist output.
- `base_state.run_id` could diverge from `ProductionEnvelope.trial_id`.
- Cost/model evidence could be recorded as `0.0` / `unknown`.
- Contribution/envelope immutability was shallow; digest canonicalization accepted arbitrary `default=str`.
- JSON Schema cannot express all runtime invariants.
- `v42` graph version was hardcoded without artifact validation.

### Edge Case Hunter

- Full scaffold nodes, including `gate_decision`, were bypassed.
- Full `production_envelope` was injected into specialist state, allowing undeclared dependency reads.
- Missing output could be silently accepted because `cache_state.cache_prefix` starts as the input payload.
- Duplicate contribution rejection happened after invocation.
- Cost/model evidence remained permissive.

### Acceptance Auditor

- **BLOCKER:** adapter skipped scaffold/gate boundary; AC-B, AC-C, Path A-prime, N2, and N6 failed.
- **HIGH:** N6 was marked PASS without gate-boundary proof.
- **HIGH:** AC-E isolation evidence needed full execution evidence, not just builder shape.
- **HIGH:** focused review diff omitted RunState schema pins; actual working tree contained the pins.
- **MEDIUM:** digest canonical JSON was under-specified and under-tested.
- **MEDIUM:** cost rollup semantics needed either explicit cost source or documentation.
