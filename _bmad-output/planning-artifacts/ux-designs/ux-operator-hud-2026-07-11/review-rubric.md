# Spine Pair Review — Operator HUD — Flight Deck

## Overall verdict

A strong, unusually disciplined spine pair: the read-only zero-button posture is enforced everywhere (interactivity budget stated exhaustively, verdict affordances banned by name), all seven envelope statuses render distinctly with verbatim vocabulary, and the invented Projection Demands section gives the architecture phase a clean UX-side contract to build against. The gaps a downstream consumer would actually hit are three medium items: the Notification toast has behavior but no visual spec, three context-panel variants (current-step briefing, deliverables summary, failure briefing) are named but never specced, and the contrast floor lives in an EXPERIENCE.md parenthetical while EXPERIENCE.md itself says contrast lives in DESIGN.md. Nothing is critical; the pair is fit to hand to architecture with those patches, and the [ASSUMPTION]-tagged thresholds are correctly flagged for the operator's UX-gate review rather than silently pinned.

## 1. Flow coverage — strong

Checked every requirement-shaped source item against Key Flows: the brief's six-phase run lifecycle, success criteria 1–5, the party plan's JTBD ranking (#1 why-stopped/what-next, #2 healthy-vs-wedged, #3 produced-so-far, #4 cost one-liner), the moment-based IA (returning / watching / deciding / closing), and Sally's six binding non-negotiables. All six flows have a named protagonist (Juan), numbered steps, and a climax beat; Flows 1–5 carry failure paths. Sally's non-negotiables all land: 5-second rule (Flow 2 + layout law), every-pause-renders-a-next-action (Flows 3/4/5 + State Patterns table), artifacts-under-judgment inline (Flow 3 step 2 + gate briefing card), urgency auto-expand verbatim (§Two-Stage Map), staleness honesty (Flow 2 failure + Zero-Lie contract), Dev-Cycle/M5 rejected (§Inspiration). All three pause flavors get their own flow; batch flow also exercises the stall watchdog.

### Findings
- **low** Flow 6 (Landing) has no failure path, and the `failed` terminal status renders only in the State Patterns table — never exercised in a flow (EXPERIENCE.md §Key Flows, Flow 6). *Fix:* add a failure beat to Flow 6 (e.g., envelope flips `failed` at node 15 → failure briefing with LangSmith link) or a two-step mini-flow for `failed`.
- **low** The brief's "drill-down everywhere" pillar (specialist icons expand to briefings; state-trace window) is specced in Component Patterns but never exercised in any flow — Flow 2 only glances at Gary's ▶ glyph (EXPERIENCE.md §Key Flows vs brief.md §Run lifecycle item 5). *Fix:* one step in Flow 2 or 4 where Juan expands a specialist chip or the state-trace well.
