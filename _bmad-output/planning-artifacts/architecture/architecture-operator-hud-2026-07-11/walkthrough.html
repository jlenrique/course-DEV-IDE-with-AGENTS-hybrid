Operator HUD — Flight Deck

Single-writer projected read model (CQRS-lite) + dumb-terminal view: the production runtime is the sole writer of one small, versioned operator-surface.json; everything HUD-side is a pure, dumb view over that document.

Commands (verdicts, recovers, resumes) never flow through this system — they stay on the existing SPOC / gate-CLI surface. The HUD is read-only, by construction and by import-linter.

03 · Contract

The projection contract

AD-1AD-3AD-4AD-5AD-16

One versioned document is the only input to HUD data collection and the notifier. If the projection doesn't say it, downstream doesn't show it.

What rides in the document — AD-1 / AD-3

  • OperatorSurfaceProjection (Pydantic, schema_version: "v1") at state/config/runs/<trial_id>/operator-surface.json
  • Everything EXPERIENCE.md demands: identity, status, steps, pre-flight, health, trace, notifications echo
  • External references (LangSmith trace, artifacts, sample audio) ride inside as plain URLs/paths — no live calls in the data path
  • The exact next-action command string per pause class — built by a function co-located with the CLI definition; L1 round-trips every emitted command through the actual CLI parser
  • Zero-lie: no run.json parse (525KB trap), no coordination.db, no bundle-gate yaml, no LangSmith call — ever

Status vocabulary — AD-5

  • Envelope status strings embedded verbatim (all seven, mixed separators preserved)
  • L1 enum set-equality test vs production_trial_envelope.py
  • Reverse tripwire: envelope status-model files added to block_mode_trigger_paths (Tier-2, party-gated)

Dual pins + additive-only evolution — AD-4

  • Pin (a): byte-identical emitted-schema pin vs committed operator-surface.v1.schema.json
  • Pin (b): producer↔consumer parity test — field presence, enum parity, round-trip, bidirectional required/optional
  • Within v1: additive-only (new optional fields). Any removal / retype / rename → version bump + new schema file beside the old
  • Two parse surfaces: strict producer model vs read_operator_surface_lenient(raw) — ignores unknown added fields, returns typed Unrecognized on unknown version/status, never raises
  • Consumers forbidden (import-linter) from strict parse — a strict HUD parse white-screens on the very evolution the contract permits

Size caps — AD-16

  • Bounded snapshot: trace ring buffer last 200 assumption · health history last 50/tile assumption
  • Target ≤128KB serialized · L1 tripwire fails loud above 256KB
  • Full operator-visible trace, if ever wanted, is a deferred sidecar with its own contract
04 · Producer

Why the assembler exists

AD-2AD-15AD-17

The adversary review's top finding: the natural choke-point can't see the whole picture. _persist_envelope is the only run.json writer and fires in both node walks — but it sees only the envelope. Asked to emit steps / pre-flight / health, it would either crash validation or write a schema-satisfying projection that lies by omission. Hence a dedicated single writer.

OperatorSurfaceAssembler — AD-15

  • The only code that writes operator-surface.json; instantiated per trial; owns both counters; one in-process lock; bounded work only
  • Walk feeds it sections as they materialize: steps on manifest composition (graph digest + node count + walk_generation/reentered_from — index regression renders as labeled re-entry, not corruption), pre-flight items as tested, health on refresh, trace on event
  • Per-stage presence rules: at registered only identity + notifications echo; once in-flight, steps and health are mandatory

Crash window: run.json is truth — AD-17

  • A crash between the run.json persist and the projection write must not leave two truths forever (HUD showing IN FLIGHT over a paused run; the pause push never firing)
  • Projection carries envelope_digest of the run.json it derives from
  • Reconcile-on-entry: every runner entry point (start, continue/resume, recover, resume-batch, status) re-emits the projection from the current envelope before doing anything else — idempotent, cheap
  • Consumers never read run.json; their guard is staleness + the watchdog. L2 fixture: skewed pair

Exactly three legal emit triggers — AD-2

  • (a) Envelope transition — assembler.emit(envelope) inside _persist_envelope (production_runner.py:221) → covers both walks (the standing two-walk gotcha, closed structurally)
  • (b) Section update — pre-flight item completion, health refresh, trace append, via the section API
  • (c) Freshness tick — periodic while in-flight, interval ≤ 2s staleness budget (transition-only emission would break the 5s UX budget on any long node)
  • The one run.json bypass (recover reenter, ~line 3501) gets routed through _persist_envelope; after that, any direct write is a defect. L1 tripwire in CI

Windows write discipline

  • Atomic temp + os.replace; verified: raises PermissionError while a reader holds the file — bounded retry ~5× with backoff
  • On exhaustion: log and skip, never propagate into the walk — next tick self-heals (run.json stays truth)
  • Consumers open-read-close in one gulp; server reads bytes then responds — streaming the live file is a defect. L1 exercises replace-under-open-reader
05 · View server

Transport: GET-only, ETag poll

AD-6

No interactive surface can grow on the HUD port. No unbounded re-parse/re-render churn.

The server

  • app/hud/server.py — FastAPI, GET routes only: / (page), /projection (JSON), /healthz
  • No POST / PUT / DELETE / WebSocket, ever — a mutation route on this app is a defect regardless of intent
  • Own port, distinct from runtime server + gate endpoint; config key hud.port assumption: 8791

Deferred, deliberately

  • SSE is a Tier-1 follow-on — only if the gate-verdict flow independently justifies a served operator API; adopting it must not change the projection contract
  • Any future actionable affordance requires a fresh party gate (governance rider on top of the GET-only structural guard)

The poll loop

  • /projection = read-then-respond byte snapshot with ETag: <schema_version>:<seq>
  • Never mtime+size — same-size rewrites inside timestamp granularity would silently skip updates
  • Route implements If-None-Match → 304 itself (web-verified: Starlette FileResponse sets ETag but never returns 304)
  • Page polls every 2–5s with If-None-Match; re-renders only on 200
  • Updates are in-place DOM replacement, section-scoped (see slide 9) — the April location.reload() pattern retires

06 · Lifecycle

Lifecycle: session-owned, identity-bound

AD-7AD-8

The pinned start sequence — AD-7

  1. Mint trial_id (+ launch_nonce)
  2. Create run dir
  3. Write the registered projection — via the assembler, so the file exists before anything can read it
  4. Launch the HUD server — dir + projection guaranteed present; child of the runtime session; explicit trial_id argument, never discovery
  5. Run pre-flight — whose healthz item is identity-checked: passes only on trial_id + nonce match with what step 1 minted. A child bind failure is a pre-flight FAIL, never a fall-through to whatever answers the port
  6. Spawn SPOC — gated on pre-flight

Server lifetime rules — AD-7

  • Zero state beyond bound trial_id + run-dir path; survives run completion while the session lives; dies with the session
  • Mid-run server death never touches the run — SPOC and the runtime are unaffected; restartable via trial hud
  • Logs to <run_dir>/hud-server.log
  • Standalone trial hud --trial-id <id> assumption serves an existing run dir read-only, post-hoc; binds an ephemeral port, refuses hud.port when taken

The wrong-run fallback is dead — AD-8

  • April bug: _query_active_run_id → newest-mtime-bundle fallback rendered the wrong run with no indication
  • Server binds one trial_id at launch; projection carries trial_id; on mismatch or ambiguity the view renders refuse-to-render — never a substitute run
  • coordination.db reader, _find_latest_bundle, bundle-gate yaml pipeline: deleted from the HUD path (retire story), not bypassed
  • L2 fixtures include wrong-server-on-port
07 · Producer

Pre-flight & heartbeats

AD-11

Runtime-executed, projection-reported. The runtime start path runs pre-flight (phase 01) and live heartbeats (phase 02 — real calls, never mocked/cached) and gates SPOC spawn on them. The HUD only visualizes; pre-flight stays authoritative with the HUD closed.

Item inventory is data, not code

  • Phase-01 items derive from the ready_for_trial check set, adapted in-runtime
  • Phase-02 heartbeats derive from the run's actual dependency set (component selection + specialist/service registry)
  • Falter-surface contract: anything whose absence would make the runtime falter is on the list
  • Regression rule (AD-13): any falter attributable to a checkable dependency that pre-flight passed is a pre-flight defect — each occurrence adds that check permanently

Per-platform quota mechanics (addendum §C)

  • OpenAI usage/costs needs a separate Admin API key env var — not the inference key
  • Gamma credits.remaining piggybacks generation polls; last-seen + staleness between polls

Never-false-green

  • Per-item results written into the projection as they complete: state, output, latency, quota reading + confidence
  • Confidence tags: direct / proxy / unknown — quota/credit readings may claim green only with direct or proxy
  • Heartbeat vocabulary splits missed (reading absent/overdue) from fail (check ran and failed) — two different alarms

08 · Notifier

Notifier: the watchdog must not die with what it guards

AD-9AD-10AD-18AD-19

Own process, shared service — AD-9

  • app/notify/ — launched by the start path but not fate-shared with the runtime session's Python process (adversary F7: it must survive the wedge classes that kill the session); exits on terminal status + grace
  • Consumes the projection file + hud-config.yaml only; failures never propagate to runner or server loop
  • Push via Apprise URL strings — primary Pushover (native APNs/FCM, reliable on locked/dozing phones, priority −2..2), fallback ntfy; mechanism swap is config, not code. Two configured push URLs only — a new channel class is a scope change
  • On-HUD visual + sound belong to the HUD page (from projection state + config echo) — the notifier never drives the browser
  • Restart semantics pinned: own state dir (state/runtime/notify/, never the run dir); paused_*/run_stalled fire while active + unacknowledged; batch_pause_resumed on observed transition only. L2: restart-mid-pause. Creds env-only

Stall watchdog: seq vs progress_seq — AD-10

  • seq = write counter — bumps on every write incl. freshness ticks; feeds the ETag
  • progress_seq + last_progress_at — advance only on walk-index change, node lifecycle, pre-flight completion, gate/pause/resume transition; ticks forbidden from bumping them (L1)
  • Watchdog binds to progress_seq exclusively; fires run_stalled when in-flight and no advance within budget assumption: 600s; waiting_for_provider_batch exempt by status
  • Producer-death detection: mtime frozen while in-flight + producer process gone → run_stalled ("producer dead"). No new heartbeat substrate — document monotonicity is the heartbeat

One derivation, one config owner — AD-18 / AD-19

  • derive_event_transitions(prev, cur) lives in the contract; notifier (push) and view (toast) both consume it — no phone-buzz-without-toast drift. L1 parity across all five classes incl. resume-that-repauses + recover-reenter
  • Five v1 event classes: paused_at_gate · paused_at_error · batch_pause_resumed · health_threshold · run_stalled
  • HudConfig: one model, one loader, one defaults constant in the contract; assembler echoes parsed config + parse status into the projection — that echo is what the HUD renders (header chip, "config unreadable — defaults active"), keeping AD-3 intact
09 · View

Reuse & retirement

AD-12

Retarget the data layer, keep the render shell. The April HUD's layers separate cleanly — don't torch ~98 passing render pins for aesthetics; don't keep legacy readers out of sunk cost.

Kept

  • render_html(data)-style pure-dict rendering + urgency auto-expand contract
  • ~98 render pins where compatible, re-pointed at projection-fed fixtures
  • collect/render stays a pure seam: data in, HTML out, no I/O inside render
  • New thin reader: read_operator_surface(run_dir) — explicit path argument (the L2 injection seam), no ambient discovery

Poll renders in place — never full-page

  • Section-scoped DOM replacement keyed by zone id, re-applying the urgency contract
  • Preserves disclosure state, scroll position, text selection — a snapshot that wipes the operator's half-selected command block is a defect
  • Full-page render is cold-load only; the April location.reload() pattern retires

Retired

  • Legacy data layer: hud_data_sources readers — import-linter forbids anything importing it
  • Dev-Cycle + M5 panels, with their pins
  • run_hud.py disposition pinned: becomes a deprecation stub (exits with a pointer to trial hud) in the retire story; tests/test_run_hud.py retires with it
  • coordination.db reader chain + _find_latest_bundle + bundle-gate yaml (per AD-8): deleted from the HUD path
  • Tier-2 manifest bump updates run_hud.py trigger-path rows in the same change

Why the legacy readers had to die

  • coordination.db production_runs is written only by the skills layer — the migrated runtime never writes it → query returns None → silent newest-mtime fallback → the April wrong-run bug
  • read_active_trial parsed 525KB of run.json discarding 99%; read_adhoc_summary made a live LangSmith call in the data path
