## 1. BETA definition

- **(b)** review ingestion report (completeness/errors)

## 2. "Error-free twice" — the binding pass/fail criterion (D4; Murat G-Q1)

Two failure classes, defined so the claim is falsifiable:

- **Class A — substrate defect** (deterministic given inputs): unhandled crash; false-negative gate reject over present content; a node input-starved by a missing producer; a gate over a vacuous artifact; any `*DispatchError` that is NOT operator-decision-bearing recurring. **Any Class-A event = FAIL.**
- **Class B — external-API / LLM variance** (non-deterministic): an LLM output that one roll fails a real guard (e.g. irene perception_source join) but a re-roll satisfies; an external service (Gamma/ElevenLabs) transient.

**A run is `error-free` iff:**
1. **Zero Class-A events**, AND
2. every Class-B event is absorbed by **bounded, automatic** recovery (no operator typing `trial recover`), within the **flake budget**: **≤1 auto-retry per external-API/LLM node, ≤3 total Class-B auto-recoveries per run.** Exceeding the budget = FAIL. An operator-manual recover = FAIL (it's a HIL-capability demo, not an autonomy demo).

**`twice` =** two consecutive `error-free` runs **on the same frozen corpus** with **no substrate code change between them** (a code change resets the counter to zero). The HIL verdicts (operator's a/b/c/e choices) are part of the run, not "intervention."

**Breadth (non-gating):** one additional cross-corpus run is recorded as generalization evidence, NOT one of the two qualifying runs.

**Corpus freeze:** `course-content/courses/tejal-apc-c1-m1-p2-trends` (the cycle-6 + Trial-4 certified lesson) is the **frozen BETA corpus**. Frozen as of this spec.

## 4. Capability specs

### 4.0 Tier-0 foundation (D2 — blocks everything; Amelia)
**`STRUCTURED-CARD-BINDING`** — the universal precondition. Today `variant_candidates`/`voice_candidates` are `[]`, `selected_*_id` is write-only, and `edit` doesn't re-route.
- **Card carries structured candidates:** `G2BCard.variant_candidates: list[VariantCandidate]` (id, slide_id, render path, label); `G4ACard.voice_candidates: list[VoiceCandidate]` (id, name, sample_url, characteristics).
- **Verdict read-path:** the resume path reads `selected_variant_id` (per-slide map) / `selected_voice_id` from the `OperatorVerdict.edit_payload`.
- **Re-route node:** a downstream binding node consumes the selection and emits only the chosen render per slide / the chosen voice. `edit` becomes load-bearing.
- **Exit:** an `edit` verdict at G2B/G4A measurably changes the downstream artifact.

### 4.1 (a) Sources + treatment
§02A composer already produces `directive.yaml` with per-source role + treatment. BETA adds the **Marcus conversational confirm**: Marcus reads the directive aloud (roles + treatment), operator confirms or annotates per source; the annotation maps to the existing `[c/e/s/x]` verbs under the hood. No new producer.

### 4.2 (b) Ingestion review
Close **T4-F1**: wire each specialist-summary "Emitted artifacts" to the bundle/output `manifest.json`; make the G1 drafted-proposal read the manifest (not the thin summary) so it stops false-rejecting present content; diagnose/de-dupe the Texas exit-10 double dispatch. Marcus surfaces a readable completeness/error report (file count, word-floor, any warnings) at G1.

### 4.3 (c) Lesson-plan review
Add a dedicated lesson-plan-review HIL pause after Irene Pass-1 (clone of the woken-gate pattern; content-free gate so it stays pack-neutral). Marcus surfaces plan_units (objective + scope decision + rationale); operator approves/annotates.

### 4.4 (e) Creative treatment — variants + voice (THE BIG LEAP, split per D7)
- **In-scope (picker mechanics):** operator tells Marcus **N** (variant count, e.g. 2–3); Gary **N-dispatches** and emits N renders/slide into `variant_candidates`; G2B surfaces them (Marcus presents per-slide options + the published Storyboard-A); operator's per-slide `selected_variant_id` **re-routes** so only the chosen render flows downstream. Voice picker identical via `voice_candidates` (candidates already exist in `voice_preview.voices`).
- **Distinctness floor (BETA-minimal):** variants must be *materially selectable* — at minimum a per-variant **non-title axis already available**: experience-profile / image-style / density knob varied per dispatch (NOT deep segmentation divergence). The operator must be able to SEE a difference and the pick must bind. Deep segmentation distinctness = deferred arc.
- **Clustering/pace:** read-only surface in BETA (Marcus shows CD/Irene's `cd_directive` settings; operator influence on these = deferred to the genuine-distinctness arc).

### 4.5 (d) Research review (review-only, non-blocking)
Wire the Epic-28 Irene→Tracy plan-lock fanout onto the **trial path** (today zero production-runner references): when Irene emits ≥1 IdentifiedGap, dispatch Tracy → `suggested-resources.yaml`; Marcus surfaces it; operator acknowledges/annotates (no auto-re-route for BETA). If wiring slips → (d) drops, twice-gate unaffected. **New deferred entry filed for the owner** (Mary D8).

### 4.6 (f) Motion-plan review (review-only, honest-artifact-gated)
A **minimal honest motion-plan producer** emits a `motion_plan` manifest node (declared clips or an explicit "no motion" with rationale) between §07 and §08; G2F reviews THAT. Marcus surfaces the plan; operator guides (accept/skip-motion). **If the honest plan producer is not feasible within the motion-review trial, (f) is carved out of the BETA claim in writing** and G2F is not woken (no theater gate).

### 4.7 SPOC — Marcus conversational surface
**Subprocess-narration over a structured seam (D3 — Winston Q3).** `app.marcus.cli` stays the runtime authority. A Marcus conversational layer: (1) starts the trial, (2) at each pause reads the decision-card + specialist evidence and **narrates it in persona with sanctum context**, (3) collects the operator's natural-language decision, (4) translates it to an `OperatorVerdict` and submits via the gate shim, (5) narrates the resume outcome. The seam = the decision-card JSON + verdict-file contract that already exists (keeps Trial-4 replayability). **Naming-collision (3 Marcuses) resolved at SPOC architecture pass:** Marcus-Persona (conversation) / `app.marcus` runtime / `pre_gate_marcus` node — documented, not renamed in BETA (rename = separate hygiene).

## 5. Pre-rehearsal quality infrastructure (D6 — must land + pass BEFORE the dress rehearsal)
1. **Live-dispatch contract test per woken gate** — asserts the node actually *passes* `gate_id` / emits its manifest node at live dispatch (with a stub HIL responder), NOT that a shim/template exists. Directly closes the class that ambushed node 07B.
2. **Integration-boundary ratchet (P5)** — a codified inventory of every node→gate and node→external-service boundary, each with a contract test that fails loud on drift. Kills the cascade.
3. **Crash-taxonomy guard** — a test asserting all known failure families (incl. `ModeMismatchError`) route to **error-pause, not crash** (closes `trial-4-modemismatch-recoverable-family`).
4. **Flake-budget instrumentation** — the runner auto-retries Class-B nodes within the §2 budget, logs each retry, and fails loud on budget exceed (so "error-free twice" is machine-checkable, not eyeballed).
