# L5 Narration Length Versus Engagement

## Purpose

This reference helps Wanda choose an episode length that matches the learning
job. Duration is a design constraint, not a target to fill.

## Three-Minute Explainer

Use this bracket when the learner needs one concept, one contrast, and one
next action. The structure should be direct: hook, definition, example, action.
Avoid multi-voice dialogue unless the contrast cannot be explained cleanly by
one narrator.

## Eight-Minute Deep Dive

Use this bracket when the learner needs a working model. The episode can carry
an opening tension, a misconception, an example, and a resolution. Dialogue can
help if the second voice represents a real operator question.

## Fifteen-Minute Story Arc

Use this bracket only when the lesson benefits from staged discovery. A longer
episode needs recurring stakes, spaced summaries, and chapter markers. Do not
expand by adding more definitions; expand by adding evidence and decisions.

## Retention Cues

1. Give the listener a reason to care in the first thirty seconds.
2. Restate the central contrast before the halfway point.
3. Use chapter markers for deep dives and story arcs.
4. Keep examples close to the learner's actual workflow.
5. End with one action, not a list of optional inspirations.

## Citation Pointers

1. Use course source material for factual claims.
2. Use operator-approved style guides for voice and tone claims.
3. Use artifact metadata for duration and format claims.
4. Use review notes for quality claims.
5. Avoid unsourced claims about universal attention spans.

## Duration Decision Checklist

1. If one misconception blocks progress, choose three minutes.
2. If the learner must compare paths, choose eight minutes.
3. If judgment evolves across scenes, choose fifteen minutes.
4. If the episode exceeds the budget ceiling, simplify the format.
5. If the duration grows without new learning value, cut the segment.

## Compression Pass

1. Cut duplicate definitions before cutting examples.
2. Replace long framing with one concrete scenario.
3. Move secondary caveats into show notes when they distract.
4. Keep only one recap per short explainer.
5. Keep deep-dive recaps tied to decisions, not sections.
6. Split a fifteen-minute draft only when each half has its own hook.
7. Treat silence as instructional time when it supports reflection.
8. Treat music as transition time, not content time.
9. Measure final duration after generated pacing, not only script length.
10. Re-run the length choice if live output drifts beyond the target band.
11. Keep the budget ceiling visible during duration decisions.
12. Prefer simple fallback when the learning job is narrow.
13. Document why the selected bracket fits the learner's task.
