# Memory System

## Sanctum location

`{project-root}/_bmad/memory/bmad-agent-wondercraft/`

## Batch-load order (on activation)

1. `INDEX.md` — wayfinding
2. `PERSONA.md` — identity + voice
3. `CREED.md` — principles + standing orders
4. `BOND.md` — operator + program context + voice roster
5. `MEMORY.md` — curated durable learnings
6. `CAPABILITIES.md` — capability routing

## Session-time writes (save-as-you-go)

- After each capability invocation, append a note to `sessions/YYYY-MM-DD.md`.
- If a decision or learning warrants durable capture, curate into `MEMORY.md` at session close.
- If operator preferences shift, update `BOND.md`.
- If persona traits emerge, update `PERSONA.md` (evolution log).

## Never-pretend-to-remember rule

Before responding to "what did we do last time?" or "what voice did we use on module X?" — ACTUALLY read the file. Do not fabricate continuity.
