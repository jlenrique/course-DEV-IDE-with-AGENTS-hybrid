# Delegation Intent Verification (PQ)

Authoritative content lives in [delegation-protocol.md](./delegation-protocol.md). This file registers the capability with its unique code.

## Summary

After a BMad writer (Paige / Sophia / Caravaggio) returns prose, Irene verifies it fulfills the **behavioral intent** specified in the delegation brief. This is a **structural handoff check**, not a quality gate.

- Does the prose move the learner toward the specified Bloom's level?
- Does it respect the format constraints (length, section structure, terminology)?
- Does it complement — not duplicate — the paired visual/audio channel?

PQ is orthogonal to Quinn-R's quality review. Quinn-R checks "is this good?"; PQ checks "does this match the instructional intent I briefed?"

## When to invoke

- After every BMad writer returns a delegated piece.
- Before assembling the piece into a lesson plan, slide brief, narration script, segment manifest, or assessment brief.
- Before escalating a writer-misalignment issue to Marcus.

## Procedure

1. Re-read the original delegation brief.
2. Read the returned prose.
3. Compare: Bloom's alignment, format compliance, pedagogical intent, downstream-consumption fit.
4. If aligned → accept and assemble.
5. If misaligned → provide specific feedback (cite the brief clause the prose missed) and re-delegate (max 2 revisions per piece, per [degradation-handling.md](./degradation-handling.md)).

See [delegation-protocol.md](./delegation-protocol.md) for the full workflow, writer selection matrix, and revision-round management.
