# Manual Animation Support (MA)

Implemented in `./scripts/manual_animation_workflow.py`. This file registers the capability with its unique code.

## Summary

Some motion segments are authored manually by the operator using Vyond, CapCut, or another manual-tool workflow — not generated by Kling. MA handles:

- Generating a tool-specific guidance brief (storyboard sketch, shot list, pacing target) the operator can execute.
- Validating the operator's imported asset (duration, codec, resolution) before it lands in the segment manifest.

## Fail-closed behavior

- Imported asset duration outside tolerance → halt with operator prompt.
- Asset resolution below deck minimum → halt.
- Missing required metadata → halt.

## When to invoke

- Pass 2 Step 3, for segments where `motion_plan.yaml` specifies `motion_source: imported` and a manual-tool brief is needed.
- Before MC runs (manual assets must pass MA validation before perception confirmation).

## Entry point

```python
# Direct module load (scripts/ is not a package on the sanctum side):
import importlib.util
from pathlib import Path

spec = importlib.util.spec_from_file_location(
    "manual_animation_workflow",
    Path("skills/bmad-agent-content-creator/scripts/manual_animation_workflow.py"),
)
mod = importlib.util.module_from_spec(spec)
spec.loader.exec_module(mod)

guidance = mod.generate_animation_guidance(slide_assignment, tool="vyond")
imported = mod.import_manual_motion_asset(asset_payload)
```

**Actual function signatures** (authoritative — see `./scripts/manual_animation_workflow.py`):

- `generate_animation_guidance(slide_assignment: dict, *, tool: str | None = None) -> dict` — tool-agnostic guidance dict per slide.
- `import_manual_motion_asset(asset_payload: dict) -> dict` — validates imported asset + returns manifest-ready entry.
- `ManualAnimationError` — raised on invalid payload.

See [pass-2-procedure.md](./pass-2-procedure.md) Step 3 and Epic 14 Story 14.5 for the manual-tool agent pattern.
