# Content Sequencing (CS)

Authoritative content lives in [pedagogical-framework.md](./pedagogical-framework.md). This file registers the capability with its unique code.

## Summary

Presentation order serves cognitive architecture:

- **Simple → complex** within a lesson.
- **Concrete → abstract** when introducing principles.
- **Known → novel** to scaffold onto prior knowledge.
- **Spiral curriculum** across modules — revisit concepts at deeper cognitive levels (e.g., Remember drug names in M1 → Apply dosage decisions in M2 → Evaluate interaction trade-offs in M3).

## When to invoke

- Pass 1 when multiple lessons in a module interrelate — sequence determines what can be assumed vs what must be reintroduced.
- Pass 1 when a lesson plan exceeds 20 slides — sequencing is probably wrong.
- Pass 2 when cluster narrative arcs need establish → tension → develop → resolve beat placement.
