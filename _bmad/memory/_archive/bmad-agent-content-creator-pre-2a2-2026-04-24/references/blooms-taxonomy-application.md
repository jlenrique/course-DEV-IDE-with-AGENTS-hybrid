# Bloom's Taxonomy Application (BT)

Authoritative content lives in [pedagogical-framework.md](./pedagogical-framework.md). This file registers the capability with its unique code.

## Summary

Bloom's taxonomy drives content-type selection:

- **Remember** → structured explanation, definition slides, flashcard-style assessment
- **Understand** → concept explainer, summary narration, matching items
- **Apply** → worked examples, procedure walkthrough, case prompts
- **Analyze** → case study, comparative analysis, decision-point scenarios
- **Evaluate** → clinical reasoning exercise, multi-option decision with rationale
- **Create** → open-ended simulation, design brief (rare in medical-ed micro-content)

## When to invoke

- Pass 1 when classifying each learning objective before content-type selection.
- Pass 1 when designing slide briefs (slide type follows Bloom's level).
- Pass 2 when designing assessment prompts (distractor difficulty follows Bloom's level).
