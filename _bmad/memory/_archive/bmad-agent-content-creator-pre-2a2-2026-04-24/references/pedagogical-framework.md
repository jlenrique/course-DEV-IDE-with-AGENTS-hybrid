# Pedagogical Framework

Irene's core knowledge base for instructional design decisions. Covers Bloom's taxonomy, cognitive load theory, backward design, and content sequencing.

## Bloom's Taxonomy — Cognitive Levels

Map every learning objective and content element to the appropriate level:

| Level | Cognitive Demand | Content Types That Serve It | Assessment Types |
|-------|-----------------|---------------------------|-----------------|
| **Remember** | Recall facts, terms, concepts | Terminology lists, labeled diagrams, definition narrations | Multiple choice (recognition), matching |
| **Understand** | Explain ideas, interpret meaning | Lecture narrations, explanatory procedures, mechanism descriptions | Short answer, explain-in-own-words, concept mapping |
| **Apply** | Use knowledge in new situations | Worked examples, procedure walkthroughs, guided practice scenarios | Case application, calculate/determine, procedure execution |
| **Analyze** | Break down, compare, distinguish | Case study dialogues, comparative analyses, data interpretation exercises | Compare/contrast, identify patterns, diagnostic reasoning |
| **Evaluate** | Judge, critique, justify decisions | Decision-point scenarios, evidence appraisal, treatment plan review | Justify a decision, critique a plan, evaluate evidence quality |
| **Create** | Design, construct, produce new work | Design prompts, synthesis assignments, clinical protocol development | Create a protocol, design a treatment plan, construct an argument |

### Application Rules

- Match content type to the Bloom's level of the learning objective — never over-teach (Evaluate content for a Remember objective wastes cognitive resources) or under-teach (Remember content for an Evaluate objective fails the learner)
- When a learning objective spans multiple levels (e.g., "Analyze and evaluate treatment options"), design a content sequence that scaffolds through the levels
- Assessment items MUST test at the same Bloom's level as the learning objective — this is the backward design contract

## Cognitive Load Theory

Design content that respects working memory limits:

### Intrinsic Load (complexity of the material itself)
- Cannot be reduced without simplifying the content
- Manage through prerequisite sequencing: teach foundational concepts before complex applications
- Break high-intrinsic-load topics into sequential chunks rather than simultaneous presentation

### Extraneous Load (poor instructional design)
- Reduce aggressively — this is wasted cognitive capacity
- Eliminate: decorative visuals, redundant text-and-narration, split attention between separated elements
- Apply: integrated formats (labels on diagrams, not in separate legends), worked examples before practice

### Germane Load (processing that builds schemas)
- Maximize through deliberate design
- Apply: self-explanation prompts, compare/contrast structures, interleaved practice
- Design for transfer: vary surface features while maintaining deep structure

### Practical Constraints

- **7±2 rule:** No more than 5-9 distinct items in a single content beat (slide, narration segment, assessment prompt)
- **Chunking:** Group related items into meaningful categories (drug classes, not alphabetical lists)
- **Scaffolding:** Build complexity gradually — introduce, practice, extend, apply
- **Dual coding:** Pair visual + verbal representations when possible (diagram + narration > text-only)
- **Slide brief density test:** If a slide brief contains more than 7 distinct concept items, split it or restructure

## Backward Design

The mandatory design sequence for all content:

### Step 1: Identify desired results
- What should learners know, understand, and be able to do?
- State as measurable learning objectives with Bloom's level tags

### Step 2: Determine acceptable evidence
- Design assessment items FIRST — before content
- Each assessment item tests a specific learning objective at its stated Bloom's level
- Assessment brief includes: question stem, correct answer, distractor rationale (why each wrong answer is plausibly wrong)

### Step 3: Plan learning experiences and instruction
- Design content that teaches what the assessment tests
- Select content types matching the Bloom's level of each objective
- Sequence content to scaffold from foundational to complex

### Why This Order Matters
Building content before assessment leads to content-assessment misalignment — the most common and costly instructional design failure. Backward design ensures every content element has a purpose.

## Content Sequencing

### Within a Lesson

Standard pedagogical sequence (adapt to content, not rigid):

1. **Hook/Opener** — Activate prior knowledge, create relevance. Why does this matter clinically?
2. **Concept Introduction** — Present new knowledge clearly. Define terms, explain mechanisms.
3. **Elaboration** — Deepen understanding. Examples, cases, applications.
4. **Application/Case** — Learners apply knowledge. Case study, scenario, worked problem.
5. **Synthesis** — Connect to broader context. How does this fit the bigger clinical picture?
6. **Assessment** — Verify learning. Formative check aligned to objectives.

### Across Modules (Spiral Curriculum)

- Revisit concepts at increasing complexity across modules
- Module 1: Introduce drug classes (Remember/Understand)
- Module 3: Compare drug interactions (Analyze)
- Module 5: Evaluate treatment protocols using multiple drug classes (Evaluate)

## Instructional Analysis Process

When analyzing source material (Notion notes, Box files, user descriptions):

1. **Identify content elements** — Concepts, procedures, case narratives, data, assessment opportunities
2. **Map to learning objectives** — Which objective does each element serve?
3. **Flag orphans** — Content without a learning objective is either misplaced or unnecessary
4. **Classify by Bloom's level** — What cognitive level does each element operate at?
5. **Determine content types** — Which artifact type best serves each element?
6. **Assess cognitive load** — Can a learner process this in the planned presentation format?
7. **Sequence** — Arrange for progressive cognitive demand
8. **Design assessment** — Backward design: what evidence shows learning occurred?
